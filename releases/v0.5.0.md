## FRANK MANGA+ v0.5.0

Proper Linux desktop integration.

### What changed

Up through v0.4.0 the generated `.desktop` entry shipped with empty `Categories=`, no `GenericName`, no `Keywords`, no `StartupNotify`. That meant the launcher didn't show up under any application menu category, and "manga" / "comic" / "reader" searches in your menu didn't surface it.

v0.5.0 ships a real `.desktop` file:

```ini
[Desktop Entry]
Categories=Graphics;Viewer;
Comment=Personal-use desktop reader for MANGA Plus
Exec=mangaplus-desktop
GenericName=Manga Reader
Icon=mangaplus-desktop
Keywords=manga;comic;reader;shueisha;mangaplus;jump;
Name=FRANK MANGA+
StartupNotify=true
StartupWMClass=mangaplus-desktop
Terminal=false
Type=Application
```

- Lists under **Graphics** (alongside image viewers) — wherever your DE puts that.
- Searchable by `manga`, `comic`, `reader`, `shueisha`, `mangaplus`, `jump`.
- "Manga Reader" generic name so users hovering over the icon see what kind of thing it is.
- `StartupNotify=true` — the cursor changes to "loading" while the WebView spins up.
- Wired through the `.deb`, `.rpm`, and `.AppImage` bundles uniformly.

No other code changes from v0.4.0.
