## FRANK MANGA+ v0.4.0

Polish + code health pass. One small user-visible bug fix, the rest is internal cleanup that should make the project easier to maintain going forward.

### What's new for users

- **Title cover stops getting cropped.** The title detail page now uses the proper portrait-aspect art (matching what the library and search grids show) instead of squeezing a landscape promo image into a 2:3 frame.
- **GitHub link in the top bar.** Right-aligned icon opens the repo in your system browser.
- **Multi-language URL plumbing.** Library cards already encoded each title's own language as a `?lang=` URL param; the title detail and reader pages now actually read it. Opening a Portuguese title from the library renders in Portuguese instead of falling back to English.

### Under the hood

- Wrote the missing `docs/android-secret.md` (referenced from 4 places but never existed).
- Rewrote the first-run dialog wording to match the post-v0.3.0 reality — it now appears only when auto-register fails, so it explains that instead of assuming you came in pre-set on subscriber mode.
- New `<TitleCard />` Svelte component shared by library + search; new `lib/lang.ts` is the single source of truth for locale defaults and the language-enum mapping.
- Three places hand-rolled `replace('https:', 'mpimg:')` instead of using the existing `proxied()` helper — switched them all over.
- Stripped 8 stale debug `console.log` calls from the title page and 8 stale `probe_*` debug examples from `api/examples/`.
- Slimmed `types.ts` to declare only the fields the UI reads.
- 22 new tests: 10 Rust (register handshake math pin, throttle, cache path, content-type, secret resolution), 13 TS via vitest (proxied, lang, readState).
- CI now runs the desktop crate's unit tests and vitest in addition to the existing fixture tests; clippy now covers the test modules too.
