## FRANK MANGA+ v0.7.2

Page indicator + progress bar are now per-chapter, not session-global.

### What was broken

The header indicator and footer progress bar both displayed `currentPage / loadedPages.length`. `currentPage` was a running count across *all* loaded chapters, and `loadedPages.length` was the sum of every chapter that had been auto-prefetched in the session.

So if chapter A had 108 pages and you crossed into chapter B (43 pages, just auto-loaded), landing on page 1 of B showed up as **"109 / 151"**. The progress bar filled to ~72%. The user just started a chapter — the UI said they were three-quarters of the way through it.

### What changed

Added derived state for the chapter the user is *actually* reading:

- `currentChapterFirstIndex` — index in `loadedPages` of the current chapter's first page
- `chapterPageCount` — count of pages in the current chapter only
- `pageInChapter` — 1-indexed page within the current chapter

The header indicator now shows `pageInChapter / chapterPageCount` (and `pageInChapter-(pageInChapter+1) / chapterPageCount` in double-page mode). The progress bar fills based on the same ratio. Crossing a chapter boundary resets the counter to 1/N and the bar to its starting position.

Per-chapter resume continues to work — that code was already converting to chapter-local before saving, but it now uses the same `pageInChapter` derived state for consistency.
