## FRANK MANGA+ v0.3.0

Auto-register on first launch. No phone, no ADB, no rooting required for free-tier reading.

### What changed

- On first launch with no `deviceSecret` configured, the app calls the official `/api/register` endpoint and is issued a fresh free-tier session. The returned secret is saved to the config file and reused on every subsequent launch.
- Paid subscribers keep the existing flow: extract the `deviceSecret` from a phone install and paste it into Settings (or set `MANGAPLUS_SECRET` in the environment). Either path overrides the auto-registered one.
- To go back to the free tier from a subscriber session, delete the secret file and relaunch — the app re-registers a new free-tier device.

### What you get with each path

| Setup | Catalog browsing | Free chapters | Subscription chapters |
|---|---|---|---|
| Auto-register (default) | ✅ | ✅ | ❌ |
| Pasted subscriber secret | ✅ | ✅ | ✅ |

### How the handshake works

The official Android app derives two values from `Settings.Secure.ANDROID_ID` and hits `PUT /api/register`. The salt that goes into the security key is a plaintext constant in the app (no obfuscation, no native lib, no runtime decryption). FRANK reproduces the same handshake using 8 random bytes in place of an `ANDROID_ID`. There is no DRM bypass — subscription content remains locked behind Google Play Billing receipt validation that only the official mobile app can produce.
