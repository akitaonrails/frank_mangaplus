## FRANK MANGA+ v0.6.7

Fix for the bogus end-of-title indicator on mid-title chapters.

### What was broken

The reader's auto-advance and the manual "Load next chapter" + "end-of-title" indicators all read from `MangaViewer.chapters` — the chapter list field on the `manga_viewer_v3` API response. That field looked like the full title's chapter list at a glance, but isn't.

Confirmed with a live probe against Kaiju No. 8 chapter 54:

```
=== title_detail ===
chapter_list_v2: 150 entries
first 3 chapter ids: [(1007603, "#001"), (1007604, "#002"), (1007605, "#003")]
last 3 chapter ids:  [(1025957, "#129"), (1024792, "#128"), (1024791, "#127")]

=== manga_viewer_v3 for chapter 54 (id=1012576) ===
viewer.chapters: 3 entries
viewer chapter ids: [1007603, 1007604, 1007605]
```

`title_detail` returns the full 150-chapter sequence. `manga_viewer_v3` returns only the first 3, regardless of which chapter you're reading. So `nextChapterIdAfter(1012576)` searched `[1007603, 1007604, 1007605]`, didn't find the current chapter, returned null — and the reader showed "You've reached the end" on chapter 54 of a series that has 129 chapters.

### What changed

When the reader mounts, it kicks off `get_title_detail` for the title in parallel with the page load. As soon as that response arrives (typically ~1 s after pages start streaming), the chapter list is swapped from the truncated `viewer.chapters` to the canonical full list from `chapterListV2` (falling back to `chapterListGroup` for the segmented case).

Auto-advance and the end-of-title indicator now both consult the canonical list. Chapter 54 → chapter 55 works. Same fix applies to every series where the user is past chapter 3 — Kaiju No. 8 was just the most visible symptom.

The viewer chapter list is still used in the first second before `title_detail` returns, so initial nav within the first few chapters works immediately without waiting.

### Other

- Also threads the `lang` URL param into the reader so the new `get_title_detail` call uses the same language tuple as the title page that linked here.
