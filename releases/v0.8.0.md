## FRANK MANGA+ v0.8.0

Reader features + reliability batch — six commits, all individually tested.

### New features

**Home / End jump keys.** Press <kbd>Home</kbd> to jump to the first page of the chapter you're reading, <kbd>End</kbd> for the last. Works in every layout mode.

**In-app help modal.** Press <kbd>?</kbd> or click the new question-mark button in the header to open a list of every keybinding and the click-zone layout. Auto-opens once on the user's very first chapter (latched via `localStorage` so it doesn't keep appearing).

**Broken-image recovery.** If a page fails to load — the CDN sometimes drops a request mid-fetch — the reader now does up to 3 silent auto-retries with a short delay between attempts. Most transient blips clear without you noticing. Only after the auto-budget is exhausted does the empty slot show a `↻ Reload page N` button overlay; press <kbd>R</kbd> to retry every failed image in the chapter at once.

### Reliability + perf

**Chapter-boundary advance lands on page 1 of the new chapter, every time.** Two root causes fixed:

- `advance()` used to scroll to `currentGroup + 1` *after* awaiting the prefetch. `currentGroup` can drift during the wait if the `IntersectionObserver` re-fires, so the post-await scroll targeted whatever-stale-position + 1. Now `advance()` looks up the new chapter's first group by ID via a new pure helper (`firstGroupOfChapter`), which is immune to that drift.
- Rapid tapping at a boundary queued multiple in-flight `advance()` calls, each with its own stale lookup. New re-entrancy guard drops duplicate calls while one is in progress.

**Night-mode filter no longer makes page-flip animation sluggish.** Filter is now applied to each `.manga-page` individually instead of to the whole `.page-stack`. Each image's filtered output is cached on the img itself; transforms on the stack move already-filtered surfaces around without re-rasterising the entire scroll content per frame.

**Earlier background prefetch.** `PREFETCH_TRIGGER_DISTANCE` bumped from 2 → 5. Gives the network roundtrip more time to finish before the user hits the chapter boundary.

### Tests + refactor

Pure logic for every keybinding, chapter-boundary lookup, and group-finding helper lives in `lib/readerLogic.ts` with `lib/readerLogic.test.ts` coverage. Six new vitest cases landed this release:

- Home / End → chapter-jump actions
- `?` → open-help
- `R` / `r` → reload-images
- `firstGroupOfChapter` for single mode, double mode, missing-chapter, and the regression case (108-page chapter then 43-page chapter, land on page 109)

49 vitest cases passing.
