## FRANK MANGA+ v0.6.2

UX fix for first-launch users on the free-tier auto-register path.

### What was broken

Through v0.6.1 the startup flow was:

```
no env + no file  →  auto-register a fresh free-tier device
                  →  write the new secret to the config file
                  →  app proceeds silently with the free-tier session
```

The setup dialog only ever appeared when no secret was available at all, so users who got auto-registered (most first-time launches) had no in-app indication that they were on a free-tier session. The Library was empty, subscription chapters were locked, and the only way to learn what was happening was to read the install docs.

### What changed

The backend now tracks whether the current secret came from auto-register via a marker file at `~/.config/mangaplus-reader/auto-registered` (Linux/macOS) or `%APPDATA%\mangaplus-reader\auto-registered` (Windows). The marker is created when `auto_register_secret` succeeds, cleared when the user either pastes a subscriber secret or dismisses the prompt.

The setup dialog now handles a third state. After a successful auto-register, you'll see:

> **You're reading on a free-tier session**
>
> FRANK MANGA+ couldn't find a subscriber deviceSecret on disk or in the environment, so it registered a fresh free-tier device for you automatically. You can browse the full catalog and read the chapters Shueisha offers for free — but subscription-only chapters will be locked.
>
> If you already pay for MANGA Plus on a phone, paste your subscriber deviceSecret below to unlock everything that subscription covers.
>
> `[ Continue with free tier ]` &nbsp; `[ Save & reload ]`

"Save & reload" applies the pasted subscriber secret. "Continue with free tier" dismisses the prompt — you stay on the auto-registered session and won't see this dialog again until the secret is cleared.

### Existing installs

If you already have a subscriber secret on disk (no auto-register marker present), nothing changes — no dialog pops up. The new prompt only fires on fresh installs that took the auto-register path.

### Other

`probe_free.rs` example removed — it was a one-shot diagnostic from this release's investigation.
