## FRANK MANGA+ v0.5.2

Linux-only fix. Other platforms behave identically to v0.5.1.

### What was broken

On a lot of Wayland setups (Hyprland, Sway, recent GNOME, KDE on Mesa or NVIDIA), launching FRANK MANGA+ crashed during startup with:

```
Could not create GBM EGL display: EGL_SUCCESS. Aborting...
```

That's WebKitGTK's DMABUF/EGL renderer path failing during initialization. Falling back to its SHM path works reliably across every combination I've seen.

### What changed

The binary now sets `WEBKIT_DISABLE_DMABUF_RENDERER=1` itself at the very start of `run()`, before any GTK or WebKit code touches anything. Wired into the Rust entry point so every install path (`.deb`, `.rpm`, `.AppImage`, AUR) gets the fix without packaging-level workarounds.

If you've already set the env var yourself (e.g. to `0` to opt back into the DMABUF path), the binary respects that and won't override.

No other code changes from v0.5.1.
