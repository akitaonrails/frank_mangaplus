## FRANK MANGA+ v0.7.5

Reader-page audit + refactor. No new user-facing features — the goal is making the reader's behaviour consistent and the regressions impossible.

### What was audited

State (`$state`, `$derived`, `$effect`), every event handler, every key binding, every async path, every set/get on reactive variables, every `scrollIntoView` call site. Findings:

1. `loadedChapterIds` was a plain `Set`, while `prefetchingChapterIds` and `failedChapterIds` were `$state(Set)`. Harmless today but inconsistent.
2. Two `queueMicrotask` calls were waiting for Svelte's reactive flush — `await tick()` is the correct primitive, since the reactive update may settle later than a single microtask hop.
3. Six places mutated `Set` state via `new Set(old).add(x)` / `delete(x)` boilerplate.
4. The keymap was a 10-condition `if/else if` chain in `onKey`. Not testable, not extensible.
5. `pageGroups`, `currentChapterFirstIndex`, `chapterPageCount`, `nextChapterIdAfter`, and `prevChapterIdBefore` were all inline in the `.svelte` file — pure logic with no unit-test coverage. That's exactly where the Kaiju "11 / 19" indicator bug and the cascading-prefetch bug lived.
6. Three derived chains (`currentChapterFirstIndex` → `chapterPageCount` → `pageInChapter`) each rescanned `loadedPages`. Consolidated into one.

### What changed

**New module `lib/readerLogic.ts`** holds the pure logic, with **`lib/readerLogic.test.ts`** covering it:

- `buildPageGroups(pages, mode)` — single / double / double-cover grouping
- `scanChapterBounds(pages, currentIndex)` — chapter-local first index + count
- `chapterIdAfter(chapters, id)` / `chapterIdBefore(chapters, id)` — preserves publication order, doesn't sort by id
- `findGroupContainingPage(groups, pageIndex)` — used by resume + pageMode toggle
- `keyToReaderAction(key)` — keymap, returns a tagged union

The reader page now imports and composes these. Inline implementations are gone.

**`loadedChapterIds`** is now `$state(new Set())`. Updates via `setWith` / `setWithout` helpers that the prefetching/failed sets also use.

**`queueMicrotask` → `await tick()`** in the resume-position scroll and the pageMode toggle.

**`onKey`** is now seven lines: lookup the action via `keyToReaderAction`, switch on it.

**19 new vitest cases** for the extracted helpers:

- `buildPageGroups`: empty input, single mode, double mode (even / odd counts), pairs-don't-cross-chapter-boundaries, double-cover mode, cover-offset-resets-per-chapter
- `scanChapterBounds`: bounds for each chapter, ex → #077 regression for the Kaiju bug, out-of-range fallback
- `chapterIdAfter` / `chapterIdBefore`: publication order preserved with non-monotonic IDs, end / unknown / empty cases
- `findGroupContainingPage`: range coverage, out-of-range
- `keyToReaderAction`: every binding (scroll, flip, toggles, escape), case sensitivity, null for unmapped keys

41 tests total now pass (22 existing + 19 new). Build clean, clippy clean.

### Behavioural changes

None deliberately. If anything moves, it's a bug — the test suite should catch it before the next release.
