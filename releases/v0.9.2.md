## FRANK MANGA+ v0.9.2

Reader robustness pass. A handful of small bugs around chapter boundaries that compounded into a confusing experience.

### What was broken

Five issues, all on the chapter-transition path:

1. **End-of-title flag flashed at the start of new chapters.** The viewer-side `chapters` list is truncated to a small window around the current chapter. When a new chapter loaded, that window briefly said "no next chapter exists" until the canonical list from `title_detail` resolved. The 🏁 "You've reached the end" block surfaced for a half-second every time, then disappeared. Confusing for first-time users.
2. **Space / PageDown didn't always advance to page 1 of the next chapter.** After a cross-chapter scroll, `currentGroup` wasn't updated until the `IntersectionObserver` settled the new frame past 50% visibility. The next keystroke fired `advance()` with the *old* `currentGroup` value, so the keyboard appeared to do nothing — only mouse-scroll worked because that updated the observer manually.
3. **Sometimes the next chapter wasn't prefetched in time.** Trigger distance was 5 pages from end. On slow connections you'd finish reading before the API responded, see "loading next chapter…" and a brief blank screen.
4. **Pages didn't stretch to viewport height on some titles.** Akane-banashi pages in particular — short images and a `.page-frame` that sized to the image rather than the viewport meant scroll-snap couldn't settle cleanly. Tops of the next pages peeked at the bottom of the screen.
5. **Image placeholders collapsed when the proto returned zero dimensions.** Rare, but when it happened the wrapper was 0×0 until bytes arrived, so the layout shifted noticeably as each image loaded.

### Fixes

- **`titleDetailLoaded` flag** gates the end-of-title block. While the canonical list is still in flight, a soft "checking for more chapters…" indicator shows instead. End-of-title only renders once we know the catalog has nothing more.
- **`goToGroupIndex` now sets `currentGroup` immediately**, before the smooth scroll completes. The observer still reconfirms it on settle, but the keyboard no longer races the scroll animation.
- **`PREFETCH_TRIGGER_DISTANCE` bumped from 5 to 8.** Three extra pages of reading buffer for slow API responses. No downside — prefetched chapters that go unread cost a few hundred KB of cache.
- **`.page-frame` is now `min-height: calc(100vh - 88px)` with flex centering.** Each frame reserves the full viewport, scroll-snap settles cleanly, dynamic on resize because it's in `vh`. The `.manga-page` keeps its intrinsic aspect ratio via `max-height` of the same value.
- **Image `width`/`height` attrs fall back to 836×1200** (typical MANGA Plus page) when the proto returned zeroes. Wrapper reserves the right space before bytes arrive.

### Per-commit log

Single commit — these issues are all on the same chapter-boundary code path and reviewing them together helps.

### Backwards compatibility

Pure frontend changes. No effect on the v0.9.0 render-mode policy or `mark_app_ready` handshake.
