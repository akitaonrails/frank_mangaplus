## FRANK MANGA+ v0.9.5

Svelte event-system hardening release: fewer UI lockups, better retry paths, stronger race guards, and a clean local verification run.

### UI responsiveness + error handling

- Added a shared IPC timeout helper so hung Tauri calls surface as retryable UI errors instead of infinite spinners.
- Search now registers the `all_titles_refreshed` listener before catalog fetches can trigger background refreshes, and cleans up safely if navigation happens while `listen()` is still resolving.
- Title detail renders as soon as the main detail request completes; favorite state loads independently so a slow library call no longer blocks the whole page.
- Title favorite add/remove failures now show a user-visible error instead of failing silently.
- Secret setup IPC calls now use the same timeout handling and avoid treating a secondary `is_auto_registered` failure as a failed configuration check.

### Race-condition guards

- Title and reader routes now reload on same-route param changes (`/title/1 -> /title/2`, `/reader/1 -> /reader/2`) and ignore stale async completions from older params.
- Reader chapter fetches reuse an in-flight promise per chapter/locale, so a frontend timeout plus manual retry does not duplicate backend work.
- Search favorite retry timers no longer clear a newer pending retry state.
- Reader image retry timers are cleared on success, manual retry, bulk reload, route reset, and unmount.
- Page-flip animation now captures the DOM node before async waits so unmounts cannot corrupt cleanup.

### Cleanups + tests

- Fixed all `svelte-check` errors and Help modal accessibility warnings.
- Added `@types/node` so Vite config no longer needs a stale `@ts-expect-error`.
- Added focused Vitest coverage for IPC timeout formatting/behavior and favorite error-state cleanup.
- Updated `verify.sh` to warm Svelte modules before checking virtual dev CSS chunks and reject raw Svelte payloads, matching how Vite/Svelte actually serves those endpoints.
- Added `AGENTS.md` with compact repo-specific guidance for future OpenCode sessions.

### Verification

- `./verify.sh`: all green — Rust tests/clippy, Vitest, `svelte-check`, production build, and Vite dev CSS probe.
