## FRANK MANGA+ v0.6.0

Reader rebuild around manga-RTL conventions plus a new double-page layout.

### Double-page mode

Wide-monitor users can now read two pages side-by-side, the way bound manga actually opens. Toggle from the reader header (icon next to the page counter) or with the **`D`** keyboard shortcut. The preference is saved — once you pick double, every chapter you open uses double until you toggle back.

Pairs are laid out in manga right-to-left order: the first page of the pair sits on the right, the second on the left. Pairs never cross chapter boundaries — if a chapter has an odd page count, its last page stays solo before the next chapter starts.

The page indicator shows the range in double mode (`5-6 / 24`).

### Manga RTL reading direction throughout

- **Click zones** were top/bottom (top=back, bottom=forward); they're now **left/right**, with **left = forward** and **right = back**. Matches the physical motion of flipping a bound manga page.
- **Progress bar** now fills from the right edge leftward, so the consumed portion trails behind the reader's direction of travel.
- New keyboard shortcuts to match: **ArrowLeft = forward**, **ArrowRight = back**. The existing scroll-down keys (ArrowDown, Space, j, PageDown) still advance — muscle memory preserved.

### Other

- 2 new vitest cases pinning the `pageMode` persistence default and round-trip.

No backend or API changes.
