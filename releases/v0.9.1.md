## FRANK MANGA+ v0.9.1

CSS regression fix from v0.8.0.

### What was broken

The broken-image-retry commit in v0.8.0 (`c4c63cf`) wrapped each `<img>` in a `.page-image-wrapper` div so the retry-overlay button could be positioned relative to its own image instead of the whole frame. In double-pair mode this introduced a nested `max-width: 50%` cap:

- `.page-frame.is-pair .page-image-wrapper { max-width: 50% }` capped the wrapper to half the frame
- `.page-frame.is-pair .manga-page { max-width: 50% }` ALSO still applied to the img — but it now meant 50% of the *wrapper*, which is 50% of the frame, so each img rendered at 25% of frame width

With short images, pair frames stopped filling the viewport vertically, scroll-snap had nothing to settle on, and multiple pairs ended up visible at once. Reader looked like a 4-page tiled grid instead of a single spread.

### Fix

Two CSS adjustments in the reader page:

- Drop the redundant `.page-frame.is-pair .manga-page { max-width: 50% }` rule — the wrapper already constrains the image.
- Drop the `display: flex` from `.page-image-wrapper` — it was doing nothing useful and was the indirect cause of the layout being wrong.

Now wrapper caps to 50% of the frame in pair mode, the img inside fills the wrapper's full width, frames are viewport-height-fit again, and scroll-snap settles correctly on each pair. Single-page mode was never affected.

49 vitest cases still pass. Pure CSS change — no logic or state-machine touched.
