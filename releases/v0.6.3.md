## FRANK MANGA+ v0.6.3

UI fix for the search + library round-trip, plus a third page-layout mode.

### Search "+ Library" button finally works

The `<button>` for "+ Library" was nested inside the `<a class="title-card">` anchor. That's invalid HTML, and WebKit's click handling for that combination is unreliable — preventDefault on the inner button sometimes still navigated the outer anchor, so the click silently did nothing (or navigated to the title page instead of adding the favorite).

`TitleCard` now puts the action snippet as a *sibling* of the anchor, not a child. Click events on the button are no longer in tension with the anchor's navigation. The "+ Library" / "✓ Added" / "Error" feedback shows up reliably.

### Library page now has a retry button

The library page used to spin forever if `get_favorites` stalled (rate-limit, network blip, etc.). It now races the call against a 12 s timeout and surfaces a clear error with a **↻ Retry** button instead of just spinning indefinitely.

### Triple-state page layout

The reader's D key (and the layout-toggle icon in the header) used to flip between single and double. It now cycles through three:

| Mode | Layout |
|---|---|
| single | `[1] [2] [3] [4] [5] [6]` |
| double | `[1, 2] [3, 4] [5, 6]` |
| double-cover | `[1] [2, 3] [4, 5] [6, 7]` |

The third mode matches printed manga, where the cover ships as a single page and the binding starts on the next spread. Each chapter's first page is solo, then pairs resume from page 2.

Persisted in localStorage like the other reader prefs. Default stays single. Icon in the header has a distinct glyph for each of the three states.
