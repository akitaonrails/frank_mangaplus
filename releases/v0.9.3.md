## FRANK MANGA+ v0.9.3

Reader page-sizing follow-up to v0.9.2.

### What was broken

v0.9.2 made `.page-frame` viewport-fit-height (`min-height: calc(100vh - 88px)`), but `.manga-page` kept `max-height: …` — a cap, not a force. Series with shorter intrinsic page dimensions (Akane-banashi pages are ~836×900 vs the more common ~836×1200) rendered at their natural height inside an oversized frame, leaving black letterbox above and below.

### Fix

One CSS change: replace `max-height` with `height` on `.manga-page` so images scale **up** to fill the frame, not just down to cap. Width stays `auto` so aspect ratio is preserved.

```css
.manga-page {
  height: calc(100vh - 48px - 32px - 8px);  /* was: max-height */
  width: auto;
  max-width: 100%;
  …
}
```

Single-page and double-pair modes both improve; `vh` keeps it dynamic on window resize. 49 vitest pass.

### Edge cases

- **Pair mode**: the wrapper still has `max-width: 50%`, so typical aspect ratios fit comfortably. Very wide spreads (e.g. 1672×1200 at viewport 992 height) would want more than 50% width and may slightly distort — those are uncommon in practice.
- **Very narrow windows** (<500px): `max-width: 100%` may cause slight horizontal distortion. Not catastrophic.

### Backwards compatibility

Pure CSS change. No effect on render-mode policy, prefetch logic, or end-of-title gating from v0.9.0–v0.9.2.
